LLM Speedrun Summary

Final submission snapshot for the 2,000-step GPT-from-scratch speedrun.

Dev bpb2.1009
Parameters1,913,280
Training steps2,000

What changed

  • The tokenizer was replaced with a corpus-trained byte-fallback tokenizer that learns frequent UTF-8 substrings and keeps exact round-tripping.
  • The model uses tied embeddings and a slightly better capacity allocation under the 2M parameter cap.
  • The trainer now uses AdamW, warmup plus cosine decay, and gradient clipping.

Why it worked

The biggest gain came from reducing token count on mixed English/Hindi text, which let the fixed 2,000-step budget cover more raw bytes of the corpus. The optimizer changes then stabilized the run enough for the model to make use of the better tokenization.
